# Recommended AI Agents for specific skills 

To get AI agents that support the absolute latest, cutting-edge models (like **Gemini 3.1 Pro / 3.0 Flash** and **Claude 3 Opus/Sonnet / 4.5+**), the best strategy is to look for open-source repositories built on top of model-agnostic frameworks (like LiteLLM, LangChain, LlamaIndex, or AutoGen). This allows you to easily swap in the newest API keys and model strings as soon as new models are released.

Here are the best, trustworthy, and properly licensed GitHub repositories tailored to the specific agent skills you requested:

### 1. Security Agents
These agents specialize in vulnerability scanning, penetration testing, and securing AI workflows.

*   **[protectai/llm-guard](https://github.com/protectai/llm-guard)**
    *   **License**: MIT License
    *   **Why it's great**: If you are building AI applications, this is the industry-standard security toolkit. It provides an agentic layer to sanitize prompts, prevent data leakage, block prompt injections, and evaluate outputs. It is model-agnostic and highly scalable.
*   **[NVISOsecurity/cyber-security-llm-agents](https://github.com/NVISOsecurity/cyber-security-llm-agents)**
    *   **License**: MIT License
    *   **Why it's great**: A suite of autonomous cybersecurity agents built on Microsoft's AutoGen framework. Because it uses AutoGen, you can easily plug in Gemini 3.1 or Claude's newest APIs. It can automate everyday security tasks, analyze code vulnerabilities, and run security assessments.
*   **[samuelberston/vuln-agent](https://github.com/samuelberston/vuln-agent)**
    *   **License**: MIT License
    *   **Why it's great**: An LLM security agent dedicated specifically to triaging code vulnerabilities. It connects to open-source SAST/SCA tools (like CodeQL and Semgrep), passes the findings to an LLM for context-aware analysis, and suggests secure code fixes.

### 2. Debugging Agents
These agents act as autonomous software engineers, capable of reading logs, traversing directories, and fixing runtime or logical errors.

*   **[princeton-nlp/SWE-agent](https://github.com/princeton-nlp/SWE-agent)**
    *   **License**: MIT License
    *   **Why it's great**: This is currently the gold standard for autonomous AI software engineering and debugging. It achieved state-of-the-art performance on the SWE-bench benchmark. You give it a GitHub issue or a bug report, and it interacts with a terminal, searches the codebase, and writes/tests the fix. It natively supports Claude Opus/Sonnet and can easily be configured to use Google's latest Gemini models.
*   **[Kaizen-agent/kaizen-agent](https://github.com/Kaizen-agent/kaizen-agent)**
    *   **License**: Apache-2.0 License
    *   **Why it's great**: Kaizen is a dedicated CLI debugging agent meant to help test and debug code or *other* AI workflows. It automatically detects failed test cases, generates prompt/code fixes, and applies them.
*   **[Blinky Debugging Agent](https://github.com/blinkydotdev/blinky)**
    *   **License**: MIT License
    *   **Why it's great**: Inspired by SWE-agent but integrated directly into VS Code. It triangulates runtime errors using print-statement debugging and VS Code's Language Server Protocol, then uses LLM reasoning to fix the bugs.

### 3. Database Agents (SQL, Vector DBs, & Scalability)
These agents specialize in schema creation, Text-to-SQL querying, and interacting with high-scale vector databases for RAG architectures.

*   **[run-llama/llama_index](https://github.com/run-llama/llama_index)** (Specifically the **Data Agents** module)
    *   **License**: MIT License
    *   **Why it's great**: While technically a framework, LlamaIndex is the undisputed king of Vector Database and SQL management. It features purpose-built "Data Agents" capable of writing SQL to create tables, reading schemas, and routing queries to Vector DBs (like Pinecone, Milvus, or Qdrant) for scale. It has guaranteed day-one support for new Gemini and Claude models.
*   **[spron-in/osada](https://github.com/spron-in/osada)** (Open Source AI Database Agent)
    *   **License**: Apache-2.0 License
    *   **Why it's great**: OSADA is specifically built to connect to relational databases (MySQL, Postgres) and function as an autonomous Text-to-SQL agent. It explicitly supports hooking up to Gemini out of the box, is Docker-ready, and is easy to modify for enterprise scale.
*   **[deepsense-ai/db-ally](https://github.com/deepsense-ai/db-ally)**
    *   **License**: MIT License
    *   **Why it's great**: A robust Python library for translating natural language into secure, optimized database queries. It uses agents to ensure that the queries generated by models like Claude or Gemini are syntactically sound, heavily guarded against SQL injection, and consistent across database dialects.
